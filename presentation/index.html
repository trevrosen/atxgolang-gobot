Internet of Things
(n.) the network of physical objects or "things" embedded with electronics, software, sensors, and connectivity to enable objects to exchange data with the manufacturer, operator and/or other connected devices based on the infrastructure of International Telecommunication Union's Global Standards Initiative.
Connected devices are growing huge. IPv6 will be adopted because of this. Tons of things have a TCP-IP stack and know how to talk to the internet. Even more will know how to talk only to each other.

Why Go is Great for Development Boards

  • Static compilation
  • Cross compilation
  • Awesome standard library
  • Concurrency
Can compile on your development machine and upload straight to board w/ no dependency management. This is especially nice on embedded hardware. Low resource usage, nice concurrency reasoning for doing things like reacting to lots of inputs, polling, etc.

Gobot

Gobot is a Go library for fun robotics things:

  • Program drones, toys, and development boards
  • Encapsulate functionality and easily loop, create events, etc
  • Control your creations with a web-based interface
  • Easily access pinouts and hardware interfaces on supported platforms
Really cool for getting into Go if you follow the examples and have some hardware.

Made by the Hybrid Group, a California consultancy that has released robot-control frameworks in several languages. They also help make SDKs and developer portals for new Things.

These guys are really cool. Robot frameworks in JS, Ruby. They have a bunch of projects for helping kids learn to code and play with robots (KidsRuby).

Platforms

The device/interfaces that you can program with Gobot

  • Specific toys and dev boards
  • GPIO
  • I2C
  • Bluetooth LE (coming soon)

Drivers

Encapsulate notions of Platform-independent functionality

  • Implemented as a lightweight interface
  • Hold methods for doing functionality-specific things
  • Examples: buzzers, LEDs, LCDs
Here's where a lot of your development board logic would be.

Adaptors

Encapsulate the abstraction of a Platform

  • Implemented as a lightweight interface
  • Holds pinout abstraction and things like I2C specifics
  • Abstracts away things like GPIO implementation differences
  • Handles platform specific-initialization tasks
OS-to-board numbering differences in GPIO are annoying. Different Linux kernel versions handle GPIO differently.

Web Interface

Control your creations with a web interface

  • Not well-documented, but powerful
  • Easy to expose functionality of Robots through it
  • UI is React-based and fully client-side
  • "api" package provides this
